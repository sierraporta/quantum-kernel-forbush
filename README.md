# quantum-kernel-forbush

Replication code for the paper:

> **Magnitude-Dependent Quantum Advantage in Forbush Decrease Detection:
> A Quantum Kernel SVM Benchmark**  
> D. Sierra-Porta — *Astronomy and Computing* (under review, 2026)  
> Manuscript ID: ASCOM-D-26-00170

## Overview

This repository contains the full experimental pipeline benchmarking
quantum kernel SVMs against classical RBF-SVMs for the binary
classification of Forbush decrease (FD) events versus quiet-time periods,
using multivariate solar wind and neutron monitor time series.

The central finding is that quantum advantage is **regime-dependent**:
the quantum kernel outperforms the classical baseline only above a
FD magnitude threshold of ~4%, where structured IMF–GCR correlations
provide a favourable data geometry for the ZZFeatureMap kernel.

## Repository Structure

```
quantum-kernel-forbush/
├── data_loader.py            # FEID + OMNI + JUNG loading and windowing
├── feature_extractor.py      # 121-feature extraction and MI selection
├── classical_baseline.py     # RBF-SVM (full and subsampled)
├── quantum_kernel.py         # ZZFeatureMap kernel (Qiskit 1.4)
├── main_v2.py                # Single-experiment runner
├── run_experiments_parallel.sh  # Full grid via GNU Parallel (4 cores)
├── 001_DataMining_Initial.ipynb # Data acquisition and preprocessing
├── 002_Initial_Analysis.ipynb   # Exploratory analysis
├── 003_QFD_Analysis.ipynb       # Main results and figures
├── AllExperiments.csv           # Consolidated results (180 experiments)
├── Results/                     # Per-experiment CSV outputs
└── figures/                     # Paper figures (PNG + PDF)
```

## Data Sources

All input data are publicly available:

| Source | URL |
|--------|-----|
| FEID catalog (IZMIRAN) | https://tools.izmiran.ru/w/feid |
| OMNI solar wind (NASA) | https://omniweb.gsfc.nasa.gov |
| JUNG neutron monitor (NMDB) | https://www.nmdb.eu/nest/ |

Preprocessed `.pkl` files are not included in this repository and must
be generated by running `001_DataMining_Initial.ipynb`.

## Reproducing the Results

1. Run `001_DataMining_Initial.ipynb` to generate `omni_clean.pkl`,
   `jung_clean.pkl`, and `feid_clean.pkl`.
2. Run `002_Initial_Analysis.ipynb` to verify the dataset construction.
3. Execute the full experimental grid:
   ```bash
   bash run_experiments_parallel.sh
   ```
   > ⚠ This requires approximately 2–3 days on a 4-core CPU.
   > The script is resume-safe: already completed experiments are skipped.
4. Run `003_QFD_Analysis.ipynb` to reproduce all figures and tables.

All random operations use `seed=42` throughout.
